# 项目名称
Vault Clip System V1

# 项目目标
为个人 Obsidian 仓库建立一套“源池同步 + 原文一键摘录归档”系统。

本项目不是新的笔记软件，不做完整替代 Obsidian。
本项目的目标只有两个：

1. 只把小米笔记导出的内容同步到仓库内的“源池”目录
2. 在 Obsidian 中阅读原文时，选中一句话或一个自然段，立刻弹出 A/B/C/D 分类按钮，点击后自动归档到对应分类，不离开原文

# 核心原则
1. 所有数据都落为普通 Markdown 文件
2. 原始源池与人工摘录池必须严格分离
3. 同步器只能操作源池，绝不碰摘录池
4. 插件只能写摘录池，绝不改写源池原文
5. 不做全仓库覆盖
6. 不做云同步
7. 不做移动端
8. 不做 AI 自动分类

# 仓库结构
一个 Obsidian 仓库，建议目录如下：

- 源池/
  - 这里存放从小米笔记导入的原始 Markdown 内容
  - 这是“受管区域”
- 摘录池/
  - A/
  - B/
  - C/
  - D/
  - 这里存放人工筛选后的摘录
  - 这是“沉淀区域”
- assets/
  - 图片资源目录
- .vault-clip/
  - manifest.json
  - sync-log.json
  - 这里存放同步状态、映射信息、校验信息

# 同步器职责
同步器只负责：
1. 读取小米笔记导出目录
2. 对比 Obsidian 仓库中的“源池/”
3. 只同步源池内的新增、修改、删除、重命名
4. 不得修改摘录池、assets 之外的人工目录
5. 删除操作只允许发生在源池内的镜像文件，不得波及其他目录

# 同步规则
同步器必须遵守以下规则：

## 1. 只同步指定源目录
用户会指定一个外部导出目录，例如：
D:/xiaomi-export/

同步目标固定为：
仓库/源池/

## 2. 增量同步
若导出目录新增 100 个文件，则只新增这 100 个
若原有文件内容改变，则只更新这些文件
若源目录删除某文件，则只删除源池内对应镜像文件

## 3. 限定范围删除
同步器只允许删除：
仓库/源池/ 下那些来源于小米导出且现已在导出目录消失的镜像文件

禁止删除：
- 摘录池/
- 用户自建目录
- 非源池目录
- 手工整理产生的 Markdown 文件

## 4. 文件夹变更同步
若小米导出目录中的文件夹结构发生变化：
- 新增文件夹 -> 在源池中新增
- 删除文件夹 -> 只删除源池中对应空文件夹及其镜像内容
- 合并/重命名 -> 尽量按“移动/重命名”处理，做不到时退化为新增 + 删除，但范围仍仅限源池

## 5. 图片处理
V1 假设图片较少。
同步器只需要保持 Markdown 中图片引用可用。
优先保持原有 assets 路径，不做复杂的图片搬运和去重。

# 插件职责
插件只负责：
1. 在 Obsidian 中监听用户选中文本
2. 选中后弹出 A/B/C/D 分类按钮
3. 用户点击后，把选中的文本追加写入摘录池对应分类文件
4. 写入元信息：来源笔记名、路径、时间、可选标题层级
5. 可选支持“当前摘录后自动复制原文链接”

插件禁止：
- 改写原始源池笔记正文
- 改写用户手工整理内容
- 批量重构整个仓库

# 摘录输出格式
建议输出为普通 Markdown 文件。
V1 采用“每类一个总文件”或“每类一个按月文件”，二选一。

推荐先用“每类一个总文件”：

摘录池/A.md
摘录池/B.md
摘录池/C.md
摘录池/D.md

每条摘录格式示例：

## 2026-04-21 15:20
来源：[[源池/某文件名]]
路径：源池/xxx/yyy.md

> 这里是用户选中的句子或自然段

---

# 星号提取功能
增加一个命令：
“扫描源池或指定目录，提取带 * 标记的句子/段落，输出到 摘录池/星标汇总.md”

V1 不要求高级自然语言判断。
只要求按规则提取。

# 长文性能要求
这是硬性要求：

1. 至少用 3 篇代表性长文测试
2. 打开长文时不能明显卡死
3. 选中文本后按钮出现不能有明显延迟
4. 追加写入不能阻塞 Obsidian 主界面

# 明确不做
- 云同步
- 手机端
- AI 自动打标签
- 多人协作
- 视频语音处理
- 完整新笔记系统
- 替代 Obsidian 的全功能编辑器

# 验收标准
V1 通过的条件：

1. 一个仓库内同时拥有 源池/ 和 摘录池/
2. 同步器只更新源池，不触碰摘录池
3. 打开任意源池笔记，选中文字后能弹出 A/B/C/D
4. 点击后摘录能正确写入对应分类文件
5. 写入内容带最基本元信息
6. 星号提取命令可运行
7. 长文样本可打开、可选中、可归档
8. 所有结果均为普通 Markdown，可脱离插件独立存在
